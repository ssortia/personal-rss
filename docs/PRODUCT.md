# Curio — продуктовое описание

## Что это

**Curio** — SaaS-сервис для создания персонального новостного фида с AI-фильтрацией.

Сервис собирает материалы из RSS-лент и Telegram-каналов, оценивает каждую статью на соответствие интересам пользователя с помощью LLM (Groq) и формирует персональный RSS-фид из релевантных новостей. Результат можно читать в любом RSS-ридере.

---

## Проблема

Информация о том, что интересно конкретному человеку, разбросана по десяткам источников — RSS-лент, Telegram-каналов, новостных сайтов. Читать всё подряд невозможно: слишком много шума. Ручная фильтрация отнимает время. Существующие агрегаторы предлагают либо подписку по темам (слишком широко), либо алгоритм «чёрного ящика» (непредсказуемо).

Curio решает обе проблемы одновременно: агрегирует источники в одном месте и фильтрует их по явно заданным пользователем критериям.

---

## Для кого

Любой, кто регулярно следит за новостями и хочет тратить на это меньше времени без потери качества. Технический уровень не важен — интерфейс рассчитан на широкую аудиторию.

## Ключевые возможности

### Аутентификация

- Регистрация и вход по email/паролю
- Восстановление пароля через email
- _В беклоге:_ OAuth-вход через Google, GitHub и Яндекс

### Источники

- Добавление RSS/Atom-лент по URL
- Добавление публичных Telegram-каналов
- Управление источниками через веб-интерфейс (включение/отключение, удаление)

### Фильтрация

- **Глобальные настройки** — пользователь описывает интересы в произвольной форме («интересует Go, distributed systems, не интересует политика»), выбирает категории и устанавливает порог релевантности; применяются ко всем источникам по умолчанию
- **Настройки на уровне источника** — для каждого источника можно задать отдельный текст интересов, набор категорий и порог; они имеют приоритет над глобальными. Можно сбросить до глобальных в любой момент
- LLM оценивает каждую статью по шкале 0–1 и включает в фид только те, что превышают настроенный порог
- _В беклоге:_ лайки/дизлайки для обучения персональной модели

### Доставка

- **RSS-фид** — стандартный RSS 2.0 по персональному токену (`GET /feed/:token`), совместим с любым ридером (Feedly, Reeder, Inoreader и др.)
- _В беклоге:_ Telegram-бот для получения статей в личку и управления настройками

---

## Как это выглядит

1. Пользователь регистрируется в веб-интерфейсе (email/пароль)
2. Добавляет источники (RSS-ссылки и Telegram-каналы)
3. Описывает интересы: свободным текстом и/или выбирает категории — глобально и/или отдельно для каждого источника
4. Подключает персональный RSS-фид к своему ридеру
5. Получает только то, что ему интересно

---

## Модель распространения

Curio — **open source**, бесплатный SaaS. Код открыт, облачная версия работает без платных планов.

---

## Технические ограничения и допущения

- Telegram-каналы: публичные — через HTTP-scraping, приватные — через MTProto (gramjs)
- AI-фильтрация: Groq API (llama-3.3-70b / mixtral) — fast inference, бесплатный tier
- Фид обновляется по расписанию (настраиваемый интервал на источник, от 15 минут)
