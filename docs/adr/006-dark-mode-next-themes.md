# ADR-006: Тёмная тема — next-themes + class-based подход

**Статус:** Принято
**Дата:** 2026-03-03

---

## Контекст

Приложение работало только в светлой теме. Требовалась поддержка тёмной темы с:

- переключателем в хедере
- сохранением предпочтений пользователя между сессиями
- уважением системных настроек при первом открытии
- SSR-совместимостью (без flash of incorrect theme)

---

## Решение

**`next-themes`** + class-based dark mode через `@custom-variant dark` в Tailwind v4.

`next-themes` вешает класс `.dark` на `<html>`, CSS-переменные переопределяются в блоке `.dark {}` в `globals.css`. Tailwind v4 распознаёт вариант `dark:` через `@custom-variant dark (&:where(.dark, .dark *))`.

---

## Альтернативы

### A. Media-query-only (`prefers-color-scheme`)

Tailwind v4 по умолчанию использует `@media (prefers-color-scheme: dark)`. Не требует JS.

**Минусы:** нет возможности переключить вручную; пользователь не может выбрать тему в интерфейсе.

### B. CSS-only переключатель (`:has([data-theme="dark"])`)

Можно хранить выбор в `data-attribute` без JS-библиотеки.

**Минусы:** нет готовой интеграции с React, нужно писать кастомный хук с `localStorage`, дублирование кода.

### C. next-themes (выбрано)

Готовая SSR-безопасная библиотека для Next.js:

- читает `localStorage` на клиенте
- вешает класс до гидрации (через `suppressHydrationWarning`)
- поддерживает `system` как значение по умолчанию
- стандартный выбор в экосистеме shadcn/ui

---

## Последствия

- `<html>` имеет `suppressHydrationWarning` — ожидаемое расхождение SSR/CSR при смене класса темы
- CSS-переменные дублированы в двух форматах (`--color-*` для Tailwind и `--*` для `@ssortia/shadcn-zod-bridge`)
- При добавлении новых цветов нужно обновлять обе секции (`:root` и `.dark`)
- `lucide-react` добавлен как зависимость для иконок Sun/Moon
